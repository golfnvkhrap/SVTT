SVTT v1.2 — Changelog
บันทึกการเปลี่ยนแปลงและอัปเดตระลอกใหญ่ (อัปเดตล่าสุด: 11/06/2026)

New Features & Core Engines
- Bulk/Batch Translation Engine (ระบบแปลยกแผง): เพิ่มระบบกวาดแปลแถวว่างที่เหลือทั้งหมดในคราวเดียว พร้อมติดตั้งระบบหน่วงเวลาสั้นๆ (Anti-Spam Delay 250ms) ป้องกัน Server บล็อก และปุ่มเปิดคุม CancellationTokenSource เพื่อให้กด "หยุดแปลกลางคัน" ได้อย่างปลอดภัย
- Translate Selected Rows (แปลเฉพาะแถวที่เลือก): เพิ่มความยืดหยุ่นสูงสุด ให้ผู้ใช้สามารถใช้เมาส์ลากไฮไลต์คลุมหลายแถวในตารางพร้อมกัน แล้วสั่งให้ AI เจาะจงแปลเฉพาะแถวว่างในกลุ่มนั้นได้ทันที
- Translate with Custom Limit (แปลกำหนดโควตาเอง): เพิ่มระบบป๊อปอัพด่วนถามจำนวนแถว (Numeric Dialog) เพื่อให้ผู้ใช้กำหนดจำนวนข้อความที่จะส่งให้ AI แปลในรอบนั้นๆ ได้เอง (เช่น แปลแค่ 10 หรือ 50 ข้อความ) ช่วยควบคุมค่าใช้จ่ายและเซฟโควตา API Key ได้อย่างแม่นยำ
- Smart Paste From Clipboard (วางข้อมูลอัจฉริยะ): เพิ่มฟีเจอร์การแกะและวางข้อมูล รองรับการคัดลอกข้อความมาจาก Excel (Tab-Separated) หรือรูปแบบข้อความคีย์โคลอน (Key: Value) เพื่ออัปเดตคำแปลเข้าตารางโดยตรง พร้อมระบบสแกนดักจับคีย์หลุด
- Advanced Find & Replace (ระบบค้นหาและแทนที่ขั้นสูง): เพิ่มหน้าต่างค้นหาและแทนที่ข้อความอัจฉริยะแบบเจาะลึก สามารถเลือกเปิดโหมด Match Case หรือเลือกค้นหา/แทนที่คำเฉพาะฝั่งข้อความต้นฉบับ (Original) หรือฝั่งคำแปล (Translate) ได้อย่างอิสระ
- Instant Glossary Extraction (บันทึกศัพท์เข้าคลังด่วน): เพิ่มเมนูคลิกขวาบนตาราง (Context Menu Strip) เพื่อสกัดดึงคำศัพท์ภาษาอังกฤษที่เลือกไปบันทึกคำแปลไทยลงคลังเฉพาะ (glossary.json) ได้ทันทีผ่านหน้าต่างป๊อปอัพด่วน โดยไม่ต้องเปิดหน้าต่างคลังคำศัพท์ใหญ่
- Auto-Propagation (ระบบเติมคำซ้ำอัตโนมัติ): เมื่อมีการแก้ไขหรือกดแปลข้อความใดก็ตาม ระบบจะสแกนหาประโยคภาษาอังกฤษที่หน้าตาซ้ำกันทั้งม็อด และแอบกระจายคำแปลภาษาไทยตัวเดียวกันไปใส่ให้ครบทุกแถวโดยอัตโนมัติ
- Auto Save Engine & Backup Folder: เพิ่มระบบ Background Thread ทำหน้าที่บันทึกเซฟโปรเจกต์งานแปลให้อัตโนมัติในทุกๆ 5 นาที โดยแยกไปเก็บในโฟลเดอร์สำรองเฉพาะชื่อ backup project รันเบื้องหลังทำให้หน้าจอโปรแกรมลื่นไหล ไม่กระตุกหรือค้างขณะเซฟ

UI/UX Layout Overhaul & Responsive Design
- Merged GroupBox (ยุบรวมกรอบเครื่องมือ): รื้อถอนระบบแยกกรอบคลังคำศัพท์อันเก่าที่สิ้นเปลืองพื้นที่แนวนอน ยุบรวมปุ่มเปิดคลังคำศัพท์เข้ามาอยู่ภายในกรอบ เครื่องมือช่วยเหลือการแปล (AI & Glossary) ตัวเดียวกัน จัดเรียงกลุ่มปุ่มเรียงแถวหน้ากระดานคู่เป็น 3 ชั้นอย่างสมบูรณ์แบบ
- Responsive Control Panel (ระบบล็อกสัดส่วนไซด์บาร์): ปรับตั้งค่าคุณสมบัติ FixedPanel = Panel2 และล็อกขนาด Panel2MinSize = 450 ให้กับตัวแยกหน้าต่าง (splitMain) ในหน้า Designer ทำให้แผงเครื่องมือฝั่งขวาล็อกขนาดคงที่ถาวร ไม่ว่าจะย่อหน้าต่างหรือขยายจอเต็มจอ ตัวหนังสือและสวิตช์ติ๊กถูกทั้งหมดจะไม่โดนบีบอัดจนแตกหรือโดนขอบจอตัดขาดหายไปอีกต่อไป
- Active API Status Indicator (ป้ายไฟสถานะค่าย AI): ฝังกล่องข้อความป้ายสถานะ lblActiveProvider และ lblActiveStatus ไว้บนแผงควบคุมหลัก เพื่อแสดงให้เห็นทันทีว่าปัจจุบันโปรแกรมเปิดใช้งานต่อท่ออยู่กับ AI ค่ายใด (เช่น DeepL หรือ Gemini) พร้อมสัญญาณไฟเขียว 🟢 บอกความพร้อมใช้งาน และเชื่อมโยงให้ป้ายไฟนี้รีเฟรชอัปเดตชื่อค่ายตามจริงทันทีเมื่อผู้ใช้กดบันทึกปิดหน้าต่างตั้งค่า API
- Quick Option Checkboxes (สวิตช์ติ๊กถูกคุมลอจิกหน้าจอ): ฝังกล่อง Checkbox คุมระบบทำงาน 2 ตัวเด็ดลงบนแผงหลัก:
[x] ข้ามแถวที่แปลแล้ว (chkSkipTranslated): สั่งคุมระบบแปลยกแผง ว่าจะให้ข้ามข้อความที่มีคำแปลไทยอยู่แล้ว หรือจะสั่งแปลทับล้างไพ่ใหม่
[x] เลื่อนลงแถวที่แปลอัตโนมัติ (chkAutoAdvance): เมื่อ AI แปลเสร็จ หรือเมื่อนักแปลพิมพ์งานเองแล้วกดคีย์ลัด Ctrl + Enter ในกล่อง rtbTranslate ระบบจะบันทึกคำแปลลงตาราง แล้วกระโดดไฮไลต์ลงไปแถวถัดไปให้อัตโนมัติโดยที่มือไม่ต้องละไปจับเมาส์

Smart Protection & Regex Core
- Auto-Heal Game Commands (ระบบเยียวยาโค้ดเกม): เครื่องยนต์สแกนเนอร์จะคอยตรวจจับพารามิเตอร์ของระบบเกม ไม่ว่าจะเกิดจากความผิดพลาดตอนแปลมือหรือ AI พ่นคำแปลมาเพี้ยน ระบบจะทำการดึงรหัสสัญลักษณ์เหล่านั้นกลับมาล็อกตำแหน่งเดิมให้เองก่อนเซฟ
- Bulletproof % Tokenizer (สูตรสแกนเนอร์มหาเทพ): ปรับปรุงสูตร Regular Expression ขั้นสูงใน cmdPattern เพิ่มเติมเครื่องหมายขีดล่าง _ (Underscore) กลายเป็น ...|%[a-zA-Z0-9_]+) ทำให้ระบบสามารถตามล่าจับคู่และล็อกตัวแปรระบบของเกม Stardew Valley ได้ครบถ้วน 100% ไม่ว่าจะเป็นคำเดี่ยวหรือคำผสม เช่น %Tilly, %You, %NPC_Name, %item_id ตัวโปรแกรมจะปกป้องค้างค่าอังกฤษไว้เสมอ เพื่อความปลอดภัย ป้องกัน AI แปลเพี้ยน และการันตีม็อดไม่ทำเกมแครชดับตอนนำไปเล่นจริง
- DeepL XML Tag Integration: แก้ปัญหาคำพูดเขียนติดตัวแปรเกมแล้ว AI แปลข้าม โดยการสลับมาใช้ระบบตัวล็อกชั่วคราวรูปแบบ XML Tag ตัวอักษรล้วน (<c0/>, <g0/>) พร้อมส่งคำสั่งเปิดโหมดสากล tag_handling = xml ไปยัง Server ของ DeepL ทำให้ระบบแยกแปลคำดิบออกจากตัวแปรเกมได้อย่างสมบูรณ์เด็ดขาด

Bug Fixes & Refactoring
[FIX] บั๊กโปรแกรมแครช ArgumentOutOfRangeException ในโหมด Bulk: เติมคำสั่งบังคับรีเซ็ตสเกลสูงสุด pgbProgress.Maximum = 100; ไว้ที่หัวเมธอดสรุปผล ป้องกันโปรแกรมเด้งดับเมื่อระบบแปลแบบยกแผงคำเนวณฐานสเกลเปอร์เซ็นต์ส่งกลับมา
[FIX] เคลียร์โค้ดเคลียร์ทางชนดีไซเนอร์: ย้ายโครงสร้างป้ายสถานะและ Checkbox จากเดิมที่เป็นโค้ดสดสร้างแบบ Dynamic Runtime ย้ายกลับเข้ามาฝังอยู่ในคอมโพเนนต์ดีไซเนอร์หลังบ้านอย่างถูกต้อง เคลียร์ปัญหาชื่อตัวแปรชนกันจนขึ้น Error และเปิดทางให้ขยับพิกัดผ่านเมาส์ในหน้าจอออกแบบได้เต็มรูปแบบ
[FIX #1] Progress Bar Visibility: แก้ไขให้แถบความคืบหน้า (pgbProgress) เปิดปิดและแสดงผลสัดส่วนได้อย่างถูกต้องเหมาะสมเมื่อตารางมีข้อมูลแสดงผล
[FIX #2] Auto-Reset Status Bar: ปรับปรุงให้เมธอด SetUiLoadingState(false) สั่งรีเซ็ตข้อความบนหน้าปัด lblStatus กลับเป็นคำว่า "พร้อมใช้งาน" ให้เองโดยอัตโนมัติ ช่วยลดโค้ดซ้ำซ้อนในจุดต่างๆ
[FIX #3] Memory Leak Prevention: ย้ายการ Subscribe Event ปิดเมนู Dropdown ไปไว้ใน Constructor เพียงครั้งเดียว ช่วยป้องกันการเกิดปัญหาหน่วยความจำรั่วไหล (Memory Leak) จากการเปิดปิดเมนูขวาซ้ำๆ
[FIX #5] Async Race Condition Control: เพิ่มระบบ Pending Flag เข้าไปควบคุมการทำงานใน dgvTranslations_CellValueChanged เพื่อให้ทำงานสอดคล้องลื่นไหลไปในทิศทางเดียวกับ rtbTranslate_TextChanged ป้องกันจังหวะหน่วงข้อมูลตีกัน
[FIX CS0272] Status Accessor Refactoring: เปลี่ยนสิทธิ์การเข้าถึงสถานะ Status ในคลาสข้อมูลเป็น private set แล้วตัดโค้ดเขียนทับสถานะเดิมใน Main.cs ออกทั้งหมด เพื่อให้โมเดลควบคุมการสลับสถานะอัตโนมัติผ่านช่องทางเดียวอย่างปลอดภัย
- Refactoring & Optimization: ทำการแยกสมองโค้ดส่วนกลางออกเป็นเมธอดเฉพาะตัว เช่น TryParsePastedLine(), WarnIfLegacyProject() ช่วยลดการเขียนโค้ดซ้ำซ้อน และเปิดโหมด PropertyNameCaseInsensitive = true เพื่อให้ระบบแกะอ่านไฟล์ฐานข้อมูลเดิมข้ามเวอร์ชันได้อย่างราบรื่นไม่ติดขัด
